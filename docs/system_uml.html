EBA Pillar 3 Pipeline — schemat systemu (UML)

Warstwy Medallion · przepływ danych · orkiestracja · dane referencyjne

Czytanie przepływu (czerwone strzałki): EBA → BRĄZ (parser wczytuje paczkę as-is) → SREBRO (dp-kod rozszyfrowany przez DPM2 na metrykę+wymiary, wartość otypowana; walidacja kompletności i spójności) → ZŁOTO (widoki: wskaźniki, benchmark rówieśniczy, trendy) → DASHBOARD.
Szare przerywane = zasilenia z danych referencyjnych/master (słownik DPM2, wzorzec wymogów, reguły, kontekst banku/okresu). Zielone przerywane = Prefect steruje całym łańcuchem (harmonogram, retry, log).
Kluczowa zasada: pipeline NIC NIE LICZY — CET1/LCR są sprawozdane literalnie; rola systemu = wczytaj → rozszyfruj → zwaliduj → zestaw z grupą.
